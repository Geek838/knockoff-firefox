# Firefox Extension Build Summary

## Overview
Successfully converted the Chrome-only Knockoff Amazon Brand Filter extension to work on Firefox.

## Key Changes

### 1. **Compatibility Layer** (`src/compat.js` - NEW FILE)
- Maps Firefox's `browser.*` APIs to Chrome's `chrome.*` APIs
- Handles API differences between browsers
- Polyfills missing methods like `chrome.runtime.getManifest()` for content scripts
- Ensures consistent API access across both browsers

### 2. **Manifest Updates** (`manifest.json`)
- Fixed encoding issues with special characters in the extension name
- Added `browser_specific_settings.gecko` section for Firefox:
  - Extension ID: `knockoff@knockoff.shopping`
  - Minimum Firefox version: 109.0
  - Data collection permissions: none
- Added `src/compat.js` as first content script to load compatibility layer early
- Removed duplicate `scripts` array in background section

### 3. **Background Script** (`src/background.js`)
- Added compatibility layer at the top
- Added fallback for `chrome.runtime.openOptionsPage()`
- Uses `chrome.tabs.create()` as fallback to open options page in Firefox
- Handles both Chrome and Firefox API namespaces

### 4. **Content Script** (`src/content.js`)
- Replaced `chrome.runtime.getManifest().version` with hardcoded version "0.2.0"
  - Firefox doesn't allow `getManifest()` in content scripts
- Added `hideSponsoredListings()` function:
  - JavaScript fallback for Firefox (which doesn't support CSS `:has()` selector)
  - Hides sponsored listings by setting `display: none` on matching tiles
- Updated `scan()` function to call `hideSponsoredListings()`
- Updated `rescan()` function to reset sponsored hiding styles

### 5. **Options Page** (`options/options.js`)
- Added compatibility layer at the top
- Handles Firefox's `browser.*` APIs

### 6. **Stylesheet** (`src/styles.css`)
- Removed CSS rule using `:has()` selector (not supported in Firefox)
- Sponsored hiding is now handled via JavaScript

## Testing Instructions

### Load as Temporary Add-on in Firefox:
1. Open Firefox
2. Go to `about:debugging`
3. Click "This Firefox" in the left sidebar
4. Click "Load Temporary Add-on"
5. Select `manifest.json` from this directory

### Expected Behavior:
- Extension icon appears in toolbar
- Works on all Amazon domains listed in manifest
- Filters pseudo-brands based on settings
- Hides sponsored listings when enabled
- Options page accessible via toolbar or settings

## Known Limitations

1. **CSS `:has()` Selector**: Not supported in Firefox, so sponsored hiding uses JavaScript instead
2. **`getManifest()` in Content Scripts**: Firefox doesn't allow this, so version is hardcoded
3. **API Differences**: Some Chrome-specific behaviors may differ slightly in Firefox

## Compatibility

- **Firefox**: Minimum version 109.0
- **Chrome**: Still fully compatible (no breaking changes)
- **Manifest Version**: 3 (required for both browsers)
